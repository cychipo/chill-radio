Hướng dẫn cấu hình GitHub NPM Release

chill-radio sẽ publish một bản npm patch release mới bằng GitHub Actions mỗi khi có push không phải bot vào nhánh main. Hãy cấu hình GitHub và npm trước để lần release đầu không bị lỗi giữa chừng.

Trigger: push vào main Auth: NPM_TOKEN Version: tự tăng patch Release: GitHub + npm

Workflow sẽ làm gì?

  1. Cài dependencies bằng npm ci --ignore-scripts.
  2. Chạy npm run typecheck.
  3. Chạy npm run build.
  4. Chạy npm test.
  5. Chạy npm pack --dry-run.
  6. Tăng patch version trong package.json và package-lock.json.
  7. Publish lên npm bằng NPM_TOKEN.
  8. Commit version mới, tạo tag vX.Y.Z, và push về main.
  9. Tạo GitHub Release cho tag đó.
Workflow bỏ qua github-actions[bot] và commit có [skip ci] để tránh vòng lặp release vô hạn.

1. Kiểm tra quyền publish npm package

Chạy trên máy local:

npm whoami
npm view @tgiap-dev/chill-radio name version --json
Hiện tại npm view @tgiap-dev/chill-radio name version --json có thể trả npm 404 trước lần publish đầu. Hãy xác nhận npm account của bạn sở hữu hoặc publish được trong scope @tgiap-dev trước khi bật workflow.

Nếu npm chặn scope này, hãy đổi package scope trước khi push automation vào main.

2. Tạo npm automation token

  1. Mở npm account settings.
  2. Vào phần access tokens.
  3. Tạo automation token hoặc token có quyền publish.
  4. Copy token ngay lúc tạo.
Không commit token này. Không đặt token vào .npmrc trong repo.

3. Thêm GitHub secret

  1. Vào GitHub repo → Settings → Secrets and variables → Actions.
  2. Thêm repository secret:
NPM_TOKEN=<your npm token>

Workflow sẽ đọc secret này như sau:

NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

4. Bật quyền ghi cho GitHub Actions

  1. Vào GitHub repo → Settings → Actions → General.
  2. Trong mục workflow permissions, chọn read and write permissions.
  3. Lưu lại.

Workflow cần quyền này để push commit tăng version và git tag sau khi npm publish thành công.

5. Kiểm tra branch protection

Nếu nhánh main có branch protection, hãy đảm bảo github-actions[bot] được phép push release commit và tag.

Workflow sẽ chạy:

git push origin HEAD:main --follow-tags
Nếu branch protection chặn lệnh push này, npm có thể đã publish nhưng GitHub chưa có version commit/tag. Khi đó cần recover version commit/tag trước khi push release tiếp theo.

6. Validate local trước lần push đầu

npm run typecheck
npm run build
npm test
npm pack --dry-run

Smoke test package local nếu muốn:

npm pack
npm install --prefix /tmp/chill-radio-smoke --ignore-scripts -g ./tgiap-dev-chill-radio-0.1.0.tgz
/tmp/chill-radio-smoke/bin/chill-radio --help
/tmp/chill-radio-smoke/bin/chill-radio play not-a-url

Lệnh invalid URL phải in lỗi ngắn gọn cho user và exit non-zero.

7. Checklist trước khi push lần đầu

  • NPM_TOKEN đã tồn tại trong GitHub Actions secrets.
  • GitHub Actions đã có quyền read/write.
  • Branch protection cho phép bot push release.
  • Npm account publish được package @tgiap-dev/chill-radio.
  • npm pack --dry-run không chứa tests, plans, .claude, vendor, .env, và tarball cũ.
  • Bạn chấp nhận rủi ro của bản release đầu: postinstall tải GitHub latest binaries chưa pin checksum.

8. Kiểm tra sau khi workflow chạy

npm view @tgiap-dev/chill-radio version
npm install -g @tgiap-dev/chill-radio
chill-radio --help

Kiểm tra thêm trên GitHub:

  • package.json trên main đã được tăng version.
  • Tag vX.Y.Z đã tồn tại.
  • GitHub Release vX.Y.Z đã tồn tại.

Ghi chú recovery

Nếu npm publish fail, workflow sẽ dừng trước khi push version commit/tag. Hãy sửa lỗi rồi push một thay đổi mới để retry.

Nếu npm publish thành công nhưng git push hoặc tạo GitHub Release fail:

  1. Kiểm tra các version đã publish:
npm view @tgiap-dev/chill-radio versions --json
  1. Commit/tag thủ công version đã publish, hoặc tạo GitHub Release còn thiếu.
  2. Không bump tiếp nếu bạn không cố ý tạo patch version mới.

Nếu publish nhầm một version lỗi, ưu tiên dùng:

npm deprecate @tgiap-dev/chill-radio@<version> "reason"

Sau đó publish patch version đã fix. Chỉ dùng npm unpublish @tgiap-dev/chill-radio@<version> nếu npm policy cho phép và an toàn cho user.
